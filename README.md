# Personal Learning Agent

## 项目名称

《面向个人学习管理场景的 AI Agent 任务分解、工具调用与可靠性评测研究》

## 项目性质

本项目是一个人工智能课程期末考核项目，同时也是一个个人学习管理方向的实践项目。

项目围绕 AI Agent 在个人学习管理场景中的应用展开，重点研究和实现以下能力：

- 学习日志记录与结构化管理
- 历史记忆查询
- 学习任务自动分解
- 工具调用与提醒生成
- 科技热点获取与学习关联分析
- Agent 输出结果的可靠性评测



---

## 一、项目背景

在日常学习中，学生经常会遇到以下问题：

- 每天做了什么事情没有系统记录
- 学习计划容易遗忘或拖延
- 过往学习内容难以快速回顾
- 多个学习任务之间缺少合理安排
- 技术热点很多，但不知道哪些内容和自己当前学习有关

随着大语言模型和 AI Agent 技术的发展，Agent 可以通过记忆、规划、工具调用等能力，辅助个人完成学习记录、任务拆解、计划生成和信息筛选。

因此，本项目尝试设计并实现一个面向个人学习管理场景的 AI Agent 系统，用于帮助用户管理学习过程，并对 Agent 的结构化、查询、规划和工具调用结果进行可靠性评测。

---

## 二、项目目标

本项目目标是实现一个可以辅助个人学习管理的 AI Agent 原型系统。

系统最终希望支持：

1. 用户输入每日学习日志，系统自动结构化保存。
2. 用户可以查询历史学习记录，例如“我上周三做了什么”。
3. 用户输入学习目标后，系统自动拆解任务并生成学习计划。
4. 系统可以根据计划生成提醒任务。
5. 系统可以获取 AI、Java 后端、大模型、Agent、开源项目等方向的科技热点，并分析其与个人学习的关系。
6. 系统可以对 Agent 输出结果进行可靠性评测，为课程报告或论文提供实验依据。

## 三、核心功能模块

### 1. 日志记录模块

用户每天输入学习日志，例如：

```text
今天复习了计网 TCP，做了 Agent 项目的数据库设计，遇到的问题是 FastAPI 不熟，明天继续做日志接口。
```

系统将日志结构化保存，包括：

- 日期
- 今日任务
- 遇到的问题
- 明日计划
- 任务类别
- 完成状态
- 学习耗时
- 备注

该模块是整个系统的数据基础，后续的记忆查询、任务规划和可靠性评测都依赖日志数据。

### 2. 提醒模块

用户可以设置提醒任务，例如：

```text
明天上午 9 点复习计网 TCP
下午 3 点推进 Agent 论文
```

系统可以保存提醒，并在后续版本中通过定时任务触发提醒。

计划支持：

- 手动创建提醒
- 从明日计划中自动生成提醒
- 查看今日提醒
- 修改提醒状态

### 3. 历史记忆查询模块

用户可以用自然语言查询历史记录，例如：

```text
我上周三做了什么？
我前几天是不是看过 Agent 论文？
我这个月项目推进到哪了？
```

系统会先查询数据库或向量库，再结合大模型生成总结回答。

该模块体现 AI Agent 的 Memory 能力，也可以扩展为 RAG 检索增强生成。

### 4. 智能规划模块

用户输入阶段目标，例如：

```text
这周我要复习计组、做 Agent 项目、写论文开题。
```

系统根据历史日志、未完成任务和用户目标，自动拆解任务并生成学习计划。

该模块体现 AI Agent 的任务分解能力。

### 5. 科技热点模块

系统每天获取 AI、Java 后端、大模型、Agent、开源项目等方向的科技热点，并生成摘要。

每条热点需要说明：

- 新闻内容是什么
- 为什么值得关注
- 和当前学习有什么关系
- 是否建议加入学习计划

该模块体现 Agent 的外部信息获取和工具调用能力。

### 6. 可靠性评测模块

为了体现人工智能课程项目的研究价值，本项目将加入可靠性评测。

评测方向包括：

- 日志结构化准确率
- 历史记忆查询准确率
- 任务分解合理性
- 工具调用成功率

评测结果可以用于课程报告、期末展示或后续论文写作。

## 四、技术路线

### 后端

- Python
- FastAPI
- SQLAlchemy
- SQLite

### AI 能力

- 大语言模型 API
- Prompt 设计
- JSON 结构化输出
- RAG 检索增强生成
- Embedding 向量表示

### 定时任务

- APScheduler

### 数据存储

第一版使用 SQLite，便于快速开发和本地运行。

后续可以扩展为：

- MySQL
- MongoDB
- 向量数据库
- Notion 数据同步

### 前端

第一版使用简单 HTML、CSS 和 JavaScript。

后续可以根据需要扩展为 Vue 或 React。

## 十一、桌面版说明

当前项目已经新增桌面版入口：

```text
desktop_app.py
```

桌面版不需要浏览器，也不需要先启动 FastAPI 后端服务。它会直接读写 SQLite 数据库：

```text
data/learning_agent.db
```

启动方式：

```bash
python desktop_app.py
```

当前桌面版已支持：

- 新增学习日志
- 保存到 SQLite 数据库
- 查看历史日志
- 按日期筛选日志
- 按关键词筛选日志
- 查看日志详情
- 删除选中日志

桌面版当前界面设计：

- 主标题为“今天也要开开心心！”
- 日期自动显示在右上角，不需要手动填写
- 今日计划显示在右上角区域
- 今日笔记作为主要输入区域，面积更大
- 待解决事项位于今日笔记下方
- 历史日志作为左侧边栏，点击“历史”按钮后展开
- 明日计划也放入左侧边栏，点击“明日计划”按钮后展开
- 页面删除了课程说明文案，保留更简洁的日记式界面
- 整体色调改为低饱和淡粉色
- 背景使用本地动漫少女学习主题插图：

```text
assets/anime_study_background.png
```

当前项目同时保留 Web 版和桌面版：

```text
personal-learning-agent/
  app/                  # FastAPI 后端
  data/                 # SQLite 数据库
  frontend/             # Web 页面
  desktop_app.py        # 桌面版入口
  requirements.txt
  README.md
```

目前桌面版仍属于日志记录模块第一版，还没有接入大模型，也还没有实现 Agent 工具调用。后续会在此基础上继续加入 AI 自动结构化、历史记忆查询、提醒、智能规划和可靠性评测。
