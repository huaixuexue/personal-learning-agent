# Personal Learning Agent 网页版说明
# 本项目为个人项目，包括网页版和应用版，应用版比网页版欠缺了一些功能，仅仅适合个人使用。部署了轻量级服务器，可以通过http://1.14.97.233:8080/ 访问测试，查看功能。在很多地方还不够完善，会持续优化。

## 1. 项目定位

Personal Learning Agent 网页版是一个基于 FastAPI、SQLite 和浏览器前端的个人学习管理系统。它面向学习日志记录、今日计划管理、文件资料管理、AI 对话分析、历史检索和阶段记录导出。

当前网页版适合用于正式演示、论文原型展示和个人部署使用。它的核心目标是把每天的学习计划、笔记、问题、上传资料和 AI 交互统一保存下来，形成可检索、可分析、可导出的个人学习记忆库。

## 2. 当前可用功能

### 2.1 用户登录

- 支持用户名和密码登录。
- 第一次使用某个用户名登录时会自动创建账号。
- 再次登录同一用户名时需要输入原密码。
- 密码以加盐哈希形式保存，不保存明文密码。
- 不同用户名之间的数据相互隔离。

### 2.2 每日学习记录

主界面支持记录：

- 日期
- 今日心情
- 今日计划
- 今日笔记
- 待解决事项
- 明日计划
- 任务类别
- 完成状态
- 学习耗时
- 备注

页面打开时，中间日期会自动同步为当天日期。切换日期后，系统会读取该日期的最新记录并回填到页面。

### 2.3 结构化今日计划

今日计划采用“大框 + 多个小计划框”的形式：

- 大框外有“新建”和“删除”按钮。
- 点击“新建”会新增一条计划小框。
- 每条计划前会显示编号，例如 1、2、3。
- 每条计划右侧有“已完成”和“未完成”按钮。
- 点击状态后颜色会加深，用于记录计划执行情况。
- 点击某条计划可以选中，再点击删除时只删除当前选中的计划。

系统会把计划同时保存为结构化数据和兼容旧版本的文本数据。结构化数据用于后续训练 AI 读取计划执行情况，文本数据用于历史检索和导出。

### 2.4 历史搜索

历史搜索支持：

- 从今天开始显示历史记录。
- 按日期搜索日志。
- 按关键字搜索日志。
- 查看今日计划、待解决事项、今日笔记和明日计划。

历史搜索旁边提供“导出”按钮，用于按日期区间导出 Word 文档。

### 2.5 Word 导出

点击“导出”后会打开弹窗，选择：

- 起始日期
- 终止日期

点击确定后，系统会下载一个 Word 文档。文档内容包括：

- AI 阶段总结
- 区间内每天的今日计划
- 区间内每天的待解决事项
- 区间内每天的今日笔记

AI 总结会聚焦：

- 这一阶段主要做了什么事情
- 暴露了哪些问题和待解决事项
- 未来需要改进什么
- 后续应该朝哪个方向规划

当前导出不会总结“今日计划整体完成情况”。

### 2.6 AI 学习助手

AI 助手支持：

- 普通对话
- 读取历史学习日志
- 结合用户画像回答问题
- 根据“规划”“帮我规划一下”等输入生成规范计划文本
- 将 AI 生成的计划一键应用到今日计划
- 上传文件后把文件内容或文件信息交给 AI 参考分析

当用户输入中出现“规划”相关表达时，AI 会优先输出适合写入今日计划的编号式任务文本，方便点击“应用”后写入计划框。

### 2.7 AI 文件上传

AI 对话输入区旁边提供“上传”按钮。上传后，文件会先保存到后端，再作为 AI 对话上下文使用。

当前支持上传：

- 文本类文件
- Word 文档
- PDF 文件
- 图片文件
- 常见课程资料和项目文件

说明：

- 文本文件、Word、部分 PDF 可以提取文字交给 AI 分析。
- 图片文件目前可以保存和管理，但如果要让 AI 直接理解图片内容，需要后续接入 OCR 或多模态模型。
- 扫描版 PDF 如果没有可提取文字，也需要后续接入 OCR。

### 2.8 文件数据库

“数据库”按钮位于“用户画像”按钮右边，而不是用户画像弹窗里面。

文件数据库支持：

- 今日上传文件
- 历史上传文件
- 按日期分组显示历史文件
- 根据日期搜索文件
- 根据文件名关键字搜索文件
- 上传文件
- 下载文件
- 在线查看文件
- 删除文件

交互规则：

- 文件列表只显示文件名。
- 点击文件名可以选中文件，再点一次取消选中。
- 每个文件右侧有“查看”和“删除”按钮。
- 点击“查看”会在线打开文件，不会直接下载。
- 点击“删除”只删除当前文件。
- 历史文件按日期划分成不同日期框。
- 选中某个历史日期框后，可以把文件上传到该日期下。

同一个用户在不同电脑登录时，只要访问同一台服务器，就可以看到并下载自己之前上传过的文件。

### 2.9 用户画像

用户画像用于记录学习数字孪生信息，字段包括：

- 专业身份
- 研究方向
- 目标画像
- 能力状态
- 知识掌握度
- 执行习惯
- 时间约束
- 风险偏好
- 补充记忆

AI 对话时会把用户画像加入 Prompt，使回复更贴合用户的学习背景和当前目标。

## 4. 技术结构

```text
frontend/index.html      网页结构
frontend/styles.css      页面样式
frontend/app.js          前端交互逻辑
app/main.py              FastAPI 入口
app/models.py            SQLAlchemy 数据模型
app/database.py          SQLite 数据库连接
app/routers/             API 路由
app/services/            业务逻辑
assets/                  图标和背景资源
data/learning_agent.db   本地 SQLite 数据库，运行后生成
uploads/                 用户上传文件目录，运行后生成
```

## 5. 数据存储

网页版使用 SQLite 保存数据。

主要数据表：

- `user_accounts`：登录账号和密码哈希
- `learning_logs`：每日学习日志
- `plan_items`：结构化今日计划
- `ai_messages`：AI 对话记忆
- `user_profiles`：用户画像
- `uploaded_files`：上传文件记录

运行时数据位置：

```text
data/learning_agent.db
uploads/
```

这些属于本地运行数据，不应该提交到 GitHub。

## 6. AI 配置

AI Key 读取顺序：

1. `%LOCALAPPDATA%\PersonalLearningAgent\config.json`
2. 项目根目录 `config.local.json`
3. 环境变量 `PERSONAL_AGENT_API_KEY`

默认接口：

```text
https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions
```

默认模型：

```text
qwen-plus
```

示例 `config.local.json`：

```json
{
  "api_key": "你的 API Key",
  "base_url": "https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions",
  "model": "qwen-plus",
  "temperature": 0.4
}
```

`config.local.json` 包含密钥，已经加入 `.gitignore`，不要上传到 GitHub。

## 7. 本地运行

进入项目根目录：

```bash
cd personal-learning-agent
```

安装依赖：

```bash
python -m pip install -r requirements.txt
```

启动网页版：

```bash
python -m uvicorn app.main:app --host 0.0.0.0 --port 8080
```

本机访问：

```text
http://127.0.0.1:8080/
```

服务器访问：

```text
http://服务器公网 IP:8080/
```

API 文档：

```text
http://127.0.0.1:8080/docs
```
